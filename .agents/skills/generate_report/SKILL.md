# 股市投資週報生成指南

此 Skill 用於依據使用者的長期投資策略，收集近一個月重大資訊，產出可直接存放於 `reports/` 並供 Streamlit App 顯示的 Markdown 彙整報告。

## 投資核心策略

- **投資週期**：不做短線投資；買進任何標的預設持有 3 年以上。
- **選股偏好**：以價值投資、長期競爭力、前瞻產業趨勢與可驗證基本面為主。
- **市場範圍**：台股與美股為主。
- **產業範圍**：不限制產業；不可只從使用者既有 watchlist 產業延伸。必須主動掃描跨產業長期機會。
- **排除項目**：排除所有加密貨幣與加密貨幣相關消息。`ticker` 含 `-USD` 或 `tags` 包含 `加密貨幣` 的標的不得納入報告分析或配置。
- **貨幣單位**：台股以台幣 TWD，美股以美金 USD。

## 執行流程

### 1. 讀取資料

- 讀取專案根目錄的 `watchlist.json`。
- 排除加密貨幣與純現金代號，例如 `BTC-USD`、`ETH-USD`、`CASH_TWD`。
- 標記每個有效標的的狀態：
  - 持有中：`holding == true` 或 `shares > 0`
  - 僅關注：未持有但在 watchlist 中
- 檢查 `reports/` 中最新兩份既有報告，讀取配置、論點、信心指數與賣出建議，避免本期觀點無理由大幅翻轉。

### 2. 研究與來源規則

- 必須使用公開網頁搜尋或可用瀏覽工具查證近一個月資料，不能只憑記憶生成。
- 優先來源順序：
  1. 公司 IR、官方新聞稿、交易所公告、ETF 發行商、監管文件。
  2. 具編輯責任的財經媒體與研究平台，例如 Yahoo 股市、TradingView、鉅亨網、工商時報、經濟日報、Reuters、WSJ、Barron's、Investopedia、Morningstar、Tom's Hardware、ServeTheHome。
  3. 產業活動與供應鏈來源，例如 COMPUTEX、NVIDIA/Intel/AMD/MediaTek/TSMC 官方活動頁。
  4. PTT Stock、Reddit、論壇與社群只可作為「市場情緒觀察」，不得作為核心買賣理由。
- 每個重大情報都要附來源連結；若同一事件有多個來源，整併成一條情報並列出 1-3 個代表來源。
- 不要為了湊數列出無關新聞；但報告必須有「覆蓋雷達」說明哪些 watchlist 標的本期無重大新事件、為何不深入展開。

### 3. 最低完整度要求

- `關注標的重大新聞情報整理` 至少涵蓋：
  - 所有持有中的非加密標的。
  - watchlist 中評分 `rating >= 3` 的非持有標的。
  - 近一個月出現重大催化或重大風險的其他 watchlist 標的。
- 若有效 watchlist 超過 30 檔，主文至少深入分析 12 檔；其餘以「觀察池摘要」表格呈現。
- `接下來可關注的其他投資標的` 至少提出 20 檔候選，並分為：
  - 核心型：可放入長期主配置。
  - 衛星型：成長性高但波動較大。
  - 觀察型：值得追蹤但尚未到可大比例配置。
- 每個新標的需說明：投資主軸、為何現在值得看、3 年催化、主要風險、適合配置角色。
- 候選標的不得集中在單一主題。除非市場明顯缺乏機會，候選池至少覆蓋 8 個產業面向，例如：
  - AI/半導體/雲端基礎設施
  - 電網、電力設備、能源與資料中心冷卻
  - 資安、企業軟體與資料基礎設施
  - 醫療、醫材、肥胖藥、生命科學工具
  - 航太、國防、精密零組件
  - 水資源、環境服務、廢棄物處理
  - 金融基礎設施、交易所、評級與支付網路
  - 工業自動化、測試量測、物流與供應鏈
  - 消費品牌、防守型複利、必需消費
  - 台股特殊機會，例如工控、醫材、航太、電力設備、低軌衛星、內需服務
- 對每個產業面向至少列出 2-4 檔候選，並標明「為何不是只跟風題材」。
- 對潛力股要區分：
  - **高品質複利股**：護城河、現金流、定價權清楚。
  - **成長轉折股**：產業或公司基本面正在改善。
  - **早期選擇權股**：潛在上行大，但尚未適合大比例配置。

### 4. 分析格式

- 對每條情報拆成：
  - **短期觀點**：未來數週到數月，影響股價波動的方向與原因。
  - **長期觀點**：3 年以上，影響競爭力、獲利能力、護城河或產業地位的方向與原因。
- 方向只能使用：看漲、看跌、持平。
- 每個深入分析標的需列出：
  - 主要催化
  - 潛在風險
  - 追蹤指標
  - 本期結論
- 分析應避免只有摘要；每個核心標的至少要有一段「為什麼這件事對長期投資重要」的推論。
- 核心標的與新資金配置標的需補充：
  - **估值與進場紀律**：用本益比、EV/EBITDA、PS、自由現金流殖利率、同業比較或反向 DCF 其中至少一種方式描述偏貴/合理/便宜。若缺資料，需明確說明缺口。
  - **情境分析**：樂觀、基本、悲觀三情境，以及各情境下應採取的動作。
  - **催化日曆**：未來 1-2 季需追蹤的財報、產品、監管、政策或產業事件。
  - **投資組合重疊度**：指出是否與既有持股、ETF 或產業曝險高度重疊。

### 5. 新資金配置規則

- 假設有 100% 新資金，配置不限於 watchlist 內標的。
- 配置要符合長期持有與風險分散，不得把 50% 以上配置到單一個股。
- 原則上應包含：
  - 低成本大盤或市場核心 ETF。
  - 半導體/AI 基礎設施核心受益者。
  - 3-6 個跨產業高品質核心或衛星標的。
  - 5-15% 現金或短期保留資金，除非市場評價極具吸引力。
- 新資金配置不得讓單一產業主題超過 45%，並應列出大致產業曝險。若超過，需說明為何仍合理。
- 表格欄位必須包含：`標的代號`、`角色`、`建議配置比例`、`市場/貨幣`、`信心指數`、`主要理由`、`主要風險`。
- 信心指數應同時考慮：基本面可見度、估值安全邊際、競爭地位、政策/監管風險、財務品質、與目前投資策略的契合度。

### 6. 持股賣出/減碼規則

- 對所有持有中的非加密標的逐一給出：維持、加碼觀察、減碼或賣出。
- 長期策略下，不因短期雜訊建議賣出；只有出現以下狀況才建議減碼/賣出：
  - 長期 thesis 明確破壞。
  - 部位過度集中且風險報酬不佳。
  - 高估值與基本面成長不匹配。
  - 有更高信心的替代標的。
- 若建議減碼，需提供比例與執行理由；若不建議賣出，也要說明保留原因與後續觀察點。

### 7. 輸出規則

- 報告使用繁體中文 Markdown。
- 檔案存放於 `reports/report_YYYY-MM-DD.md`，日期使用本機日期。
- 若同日期報告已存在，除非使用者明確要求覆蓋，必須另存為 `reports/report_YYYY-MM-DD_主題.md`，例如 `report_YYYY-MM-DD_diversified.md`、`report_YYYY-MM-DD_cross_sector.md`。
- 報告開頭需放免責聲明：內容是研究輔助，不是保證獲利或個人化財務建議。
- 報告結尾需列出「本期資料限制」：例如來源覆蓋、未使用付費資料、未讀取雲端 Supabase 即時資料等。
- 避免 emoji 過量；可使用清楚標籤與表格提升可讀性。
